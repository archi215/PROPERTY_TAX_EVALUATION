PROPERTY_TAX_EVALUATION

Owner: archi215 - A small static project for property tax evaluation (HTML/CSS/JS + model artifacts)

Problem Statement

Build a simple property tax / price evaluation project that demonstrates dataset preprocessing, model training and a small front-end to display results and artifacts. The goal is a reproducible notebook-based pipeline that trains an XGBoost regressor, evaluates it, and saves model artifacts.

System Methodology

Dataset / Input

Expecting a CSV file with property features (example path used in the notebook: /kaggle/input/housedata/data.csv).

  • Columns used: date, price, bedrooms, bathrooms, sqft_living, sqft_lot, floors, waterfront, view, condition, sqft_above, sqft_basement, yr_built, yr_renovated, street, city, statezip, country
  • Preprocessing: drop NaNs, one-hot encode categorical columns

Model / Architecture

Trained XGBRegressor with sensible defaults (e.g. n_estimators=300, max_depth=6, learning_rate=0.05) on the features created by one-hot encoding.

Live Execution

Notebook: notebookde149eaf4a.ipynb - runs training and saves outputs: property_model.pkl, features.pkl, and predicted_prices.csv.

Results & Analysis

Example metrics from the notebook run:

  • Mean Absolute Error (MAE): ~145,500 (as produced in example run)
  • R2 Score: ~0.06 (the baseline notebook run shows this low score; further feature engineering or different models recommended)

Artifacts

  • property_model.pkl - saved model (joblib)
  • features.pkl - feature column list used for reindexing new samples
  • predicted_prices.csv - dataset augmented with predictions
  • notebookde149eaf4a.ipynb - training notebook
